1. 한눈에 보는 현재 구조

게임 코드는 기능별로 Scripts 하위 폴더에 분리되어 있고, Stage 1의 최종 런타임 데이터는 ScriptableObject asset으로 관리합니다. GameDirector는 씬 lifecycle과 서비스 조립을 담당하고, 실제 규칙/표시/액션 실행은 작은 helper와 service로 나뉩니다.

Assets/EscapeFromNightmares/
  Audio/                 - AudioMixer 등 Unity 오디오 asset
  Prefabs/UI/            - 타이틀, 인벤토리 등 UI prefab
  Resources/EscapeFromNightmares/
    Audio, CloseUps, Data, Endings, HideViews, Items, Monster, Objects, Puzzles, Rooms, Title, UI
  Scenes/                - TitleScene.unity, MainScene.unity
  ScriptableObjects/     - ResourcePathCatalog, RoomSpriteCatalog, MonsterPlacementCatalog, Stage1/
  Scripts/
    Bootstrap/ Data/ Editor/ Runtime/ Services/ UI/
  Tests/EditMode/        - Unity Test Framework EditMode tests

핵심 유지보수 기준

  • 플레이 규칙, 방/아이템/퍼즐 ID, 플래그, 리소스 경로는 ScriptableObject asset과 테스트가 기준입니다.
  • RuntimeStageFactory는 Editor 폴더에 있는 seed입니다. 런타임 코드에서 호출하지 않습니다.
  • 보이는 UI나 scene object를 추가할 때는 script만으로 끝내지 말고 prefab/scene/editor builder 반영 여부를 같이 판단합니다.
  • 저장 파일은 settings.json과 clear_records.json만 허용합니다. 진행 저장 파일을 새로 만들지 않습니다.

2. Stage 1 데이터와 로딩 흐름

Stage 1은 다음 asset 묶음으로 구성됩니다. 빌더는 기존 asset을 삭제하지 않고 필드 갱신 위주로 동작해 GUID와 scene reference를 보존하도록 설계되어 있습니다.

Stage1 rootAssets/EscapeFromNightmares/ScriptableObjects/Stage1/Stage1.asset 외 root asset 3개StageDefinition, MonsterNodeGraph, SoundCatalog.
ItemsStage1/Items - 10개인벤토리 아이템 정의.

MainScene 참조MainScene.unity의 GameDirector.stageDefinitionserialized Stage 1 asset을 우선 사용.

런타임 로딩 우선순위

GameDirector.Awake()
  -> StageRepository.LoadStage1(stageDefinition)
      1. GameDirector에 serialized StageDefinition이 있으면 사용
      2. 없으면 Resources/EscapeFromNightmares/Data/StageCatalog.asset 로드
      3. catalog.stage1 반환
  -> StageLookup 생성
  -> GameSession / services / presenters / UI bind
중요
Stage 1 데이터를 수정할 때 RuntimeStageFactory.CreateStage1()를 런타임에서 다시 쓰면 안 됩니다. 새 runtime/test 코드는 StageRepository.LoadStage1() 또는 StageTestData.LoadStage1()을 사용하세요.

3. 런타임 실행 흐름

게임 시작 후 MainScene의 GameDirector가 Stage 데이터와 서비스들을 조립합니다. 플레이어 입력은 상호작용 정의를 액션 목록으로 변환하고, Executor가 UI/세션/오디오 경계에 실제 호출을 분배합니다.

TitleSceneController.StartGame()
  -> SceneManager.LoadScene(MainScene)
  -> GameDirector.Awake()
      Stage load, lookup, services, UI reference bind
  -> GameDirector.Start()
      session.Start(stage), ambient BGM, first room render

Player click interactable
  -> EscapeActionResolver.ResolveInteractable(interactable)
  -> EscapeActionExecutor.Execute(action)
      MoveRoom / AcquireItem / OpenCloseUp / OpenPuzzle / EnterHideSpot / CompleteStage
  -> RenderRoom(), presenters, services update

5. 기능 수정 시 작업 위치

작업주로 수정할 곳주의할 점
방 추가/연결 수정Stage1/Rooms/*.asset, Stage1.asset, 필요 시 Stage1DataAssetBuilder seedroomId와 connectedRoomIds, Door targetRoomId가 서로 맞아야 합니다.
아이템 추가Stage1/Items/*.asset, 아이콘 Resources, Room interactable grantitemId는 중복 금지. iconResource가 실제 Sprite로 로드되어야 합니다.
퍼즐 추가Stage1/Puzzles/*.asset, PuzzleDefinition, PuzzlePresenter token options 필요 시 확장answerTokens, successFlag, rewardItemIds, requiredItemIds를 테스트로 검증하세요.
상호작용 추가RoomDefinition.faces[].interactablesvisible/clickable이면 scene/prefab/builder 반영 여부를 함께 판단합니다.
몬스터 배치 조정MonsterPlacementCatalog.assetenabled와 normalizedRect를 조정합니다. child_room은 기본 배치 제외.
특수 진행 규칙ProgressionFlowStage 1 고유 플래그 전환과 몬스터 시작/추격 시작 로직을 여기에 둡니다.

저장 정책 변경SettingsSaveServicesettings.json, clear_records.json 외 파일 추가는 금지된 정책입니다.

6. 테스트와 검증

EditMode 테스트는 빠른 규칙 검증과 asset 참조 검증에 집중합니다. Stage 1 테스트 데이터는 StageTestData.LoadStage1()를 통해 로드해 런타임 로딩 경로와 같은 규칙을 탑니다.

Unity.exe -batchmode -projectPath . -runTests -testPlatform EditMode -testResults TestResults/EditMode.xml

# 현재 작업 환경에서 실제 결과 XML을 생성한 명령 형태
Unity.exe -batchmode -nographics -projectPath . -runTests -testPlatform EditMode -testResults TestResults/EditMode_refactor_final4.xml
검증 항목현재 상태
StageCatalog.stage1null 아님. stageId == stage1.
중복 IDroomId, itemId, puzzleId, interactableId 중복 검증.
Room graphconnectedRoomIds와 door targetRoomId 유효성 검증.
리소스room, close-up, hide view, puzzle, item icon, sound resource 로딩 검증.
MainSceneGameDirector가 Stage1.asset을 serialized reference로 보유.
최종 테스트 결과105 total, 103 passed, 0 failed, 2 ignored. ignored 2개는 현재 worktree에 docs/ROOM_IMAGE_HARNESS.md가 없어서 문서 검증만 skip.

  • TitleScene에서 Start 버튼으로 MainScene 진입.

  • 인벤토리: 아이템 획득, 선택, 아이콘 표시 확인.
  • 퍼즐: study_safe, laundry_storage_box, breaker_box, basement_altar, front_door_escape 확인.
  • 은신: hide spot 진입/종료와 danger/noise 변화 확인.
  • 몬스터 QA 패널: 배치 상태와 enabled rect 확인.
  • Stage clear: front door escape 후 clear_records.json 기록과 title 이동 확인.

7. 인수인계 주의사항

  • 현재 일부 기존 C# XML summary comment가 문자 인코딩 문제로 깨져 보입니다. 실제 동작 이해는 본 PDF, 테스트, 타입/메서드 이름을 기준으로 보세요.
  • docs/ 및 Design/ 문서 파일들이 현재 git status에서 삭제 상태입니다. 문서 기반 테스트 2개는 이 상태 때문에 ignored 처리되어 있습니다.
  • GameDirector 안에는 아직 fallback UI 생성 코드가 남아 있습니다. 최종 화면 deliverable은 prefab/scene/editor builder를 우선하세요.
  • RoomSpriteCatalog와 ResourcePathCatalog는 직접 Sprite 참조/리소스 경로를 유지하는 핵심 catalog입니다.
  • RuntimeStageFactory는 Editor 폴더로 이동했습니다. 새 runtime/test 코드에서 직접 호출하지 말고 StageRepository 또는 StageTestData를 사용하세요.
  • 씬 참조를 잃었을 때는 Rebuild Stage 1 Data Assets와 Rebuild Main Sample Assets를 순서대로 실행해 복구합니다.
