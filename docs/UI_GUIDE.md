# UI 디자인 가이드

## 디자인 원칙
1. 화면을 가리지 않는다. UI는 게임 공간 관찰을 방해하지 않아야 한다.
2. 클릭 가능 오브젝트를 기본 하이라이트하지 않는다. 플레이어가 화면을 직접 관찰하게 한다.
3. 공포는 과장된 장식보다 어둠, 여백, 소리, 제한된 붉은색으로 만든다.

## 색상
| 용도 | 값 |
|------|------|
| 배경 | 거의 검정 `#050506` |
| 기본 선 | 흐린 회백색 `#d8d2c8` |
| 보조 텍스트 | 낮은 대비 회색 `#8c8780` |
| 위험/몬스터/제단 | 탁한 붉은색 `#8a1f1f` |
| 게임오버 | 어두운 적색 + 검정 |

## 레이아웃
- 기준 해상도는 1280x720이다.
- 상단 또는 모서리에 인벤토리, 지도, 설정 버튼을 작게 배치한다.
- 퍼즐 UI는 필요할 때만 중앙에 열고, 닫으면 즉시 플레이 화면으로 복귀한다.
- 은신 중에는 작은 위험 게이지와 최소한의 상태 표시만 보여준다.

## 컴포넌트
- 버튼: 얇은 선, 낮은 채도, 작은 텍스트
- 인벤토리 슬롯: 아이템 아이콘 + 선택 프레임
- 지도: 층별 단순 평면도, 현재 위치만 표시
- 위험 게이지: 붉은색을 사용하되 화면을 크게 덮지 않는다.
- 게임오버: 큰 연출보다 어둡게 잠식되는 느낌을 우선한다.

## 금지 사항
| 금지 사항 | 이유 |
|-----------|------|
| 네온 글로우 | 손그림풍 호러와 맞지 않음 |
| 과한 그라데이션 | 악몽의 조용한 압박을 약화함 |
| 클릭 오브젝트 상시 하이라이트 | 탐색 긴장감이 사라짐 |
| 큰 튜토리얼 문구 | 몰입을 깨뜨림 |
| 밝고 선명한 UI 팔레트 | 공포 톤과 충돌함 |

## MVP 예외
- 실제 리소스가 없을 때는 개발 검증을 위해 텍스트 기반 상호작용 목록을 임시 허용한다.
- 최종 리소스 적용 단계에서는 배경 위 normalizedHitbox 클릭 방식으로 전환한다.
