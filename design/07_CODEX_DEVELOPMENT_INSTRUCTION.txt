Escape From Nightmares
Codex Unity 최종 개발 지시서

================================================================================
01. 기본 지시
================================================================================

너는 Unity 기반 Windows PC 게임 개발 AI다.

프로젝트명:
Escape From Nightmares

Unity 프로젝트 경로:
EscapeFromNightmares/

기술 기준:
- Unity 6.3.9f1
- C#
- Universal Render Pipeline 2D
- Unity Input System
- UGUI Canvas / EventSystem / TextMeshPro

목표:
첨부된 기획 문서들을 기준으로 실행 가능한 Unity Stage 1 MVP를 제작한다.

최종 산출물:
1. Unity 프로젝트 폴더
2. Stage 1 플레이 가능한 Unity Scene
3. Windows PC player build

중요:
- 웹 게임이 아니다.
- Ren'Py, HTML5, JavaScript 게임이 아니다.
- 반드시 Unity / C# 기반 Windows PC 게임으로 제작한다.
- 기획서에 없는 설정은 임의로 확장하지 않는다.
- 리소스가 없으면 지정된 파일명 기준으로 임시 placeholder 리소스를 생성해서라도 실행 가능하게 만든다.
- 단, 구조는 최종 리소스로 교체 가능한 형태여야 한다.
- 클릭 가능한 오브젝트는 화면에 표시하지 않는다.
- 진행 저장은 절대 구현하지 않는다.
- 설정과 클리어 기록만 저장한다.

03. 완료 기준
================================================================================

완료된 결과물은 다음 조건을 만족해야 한다.

- Start를 누르면 child_room에서 시작한다.
- 인벤토리, 지도, 설정 버튼이 작동한다.
- 방 이동이 가능하다.
- 아이템 획득이 가능하다.
- 주요 퍼즐이 작동한다.
- 몬스터 첫 등장이 발생한다.
- 몬스터가 방/노드 단위로 이동한다.
- 은신 시스템이 작동한다.
- 잡히면 Game Over 후 처음부터 재시작한다.
- 지하실 제단 퍼즐을 풀면 현관 열쇠가 나타난다.
- 현관 열쇠 획득 후 최종 추격이 시작된다.
- 현관문을 열면 스테이지 클리어된다.
- 설정과 stage1_clear는 저장 가능하다.
- 진행 저장은 절대 하지 않는다.
- 최종적으로 Unity 프로젝트와 Windows player build를 생성한다.

================================================================================
04. 리소스 관련 지시
================================================================================

리소스가 없을 경우:
- placeholder Sprite 생성
- 무음 AudioClip 또는 재생 생략 fallback 처리
- 파일명과 참조 구조는 유지
- 최종 리소스로 교체 가능한 구조 유지

이미지 placeholder:
- 1280x720 어두운 배경
- 방 이름을 작게 표시해도 됨
- 클릭 오브젝트를 하이라이트하지 않음

사운드 placeholder:
- 파일이 없으면 오류가 나지 않게 fallback 처리
- 무음 AudioClip 또는 재생 생략 처리 가능
