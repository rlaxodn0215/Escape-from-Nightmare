================================================================================
01. 현재 부족한 부분
================================================================================

현재 기획서와 개발 지시서는 스테이지 1을 구현하기 위한 핵심 구조를 대부분 포함하고 있다.
다만 실제 개발 단계에서 아래 항목은 추가 확정 또는 실제 리소스 작업이 필요하다.

1. 클릭 오브젝트 좌표
- 모든 클릭 오브젝트는 사각형 hitbox를 사용하기로 확정했다.
- 하지만 실제 x, y, w, h 좌표는 최종 방 배경 이미지가 있어야 정확히 정할 수 있다.
- 현재 단계에서는 임시 좌표 또는 더미 좌표를 사용해야 한다.

2. 실제 방 배경 이미지
- 방 배경 파일명과 아트 방향은 확정했다.
- 실제 이미지 리소스는 아직 없다.
- Codex 구현 시에는 더미 이미지로 먼저 실행 가능하게 만들고, 이후 최종 리소스로 교체 가능한 구조가 필요하다.

3. 실제 오브젝트 이미지
- 아이템, 퍼즐 장치, 문, 은신 장소, 몬스터 연출 이미지 파일명은 정했다.
- 실제 이미지 리소스는 아직 없다.
- 더미 오브젝트 또는 단순 도형/텍스트 placeholder로 구현 가능해야 한다.

4. 실제 사운드 파일
- 사운드 파일명과 카테고리는 정했다.
- 실제 ogg 파일은 아직 없다.
- 없을 경우 무음 처리 또는 더미 사운드로 대체해야 한다.

5. 몬스터 등장 연출 세부 프레임
- 몬스터 외형과 등장 방식은 확정했다.
- 각 등장 연출의 이미지 프레임 수, 지속 시간, 애니메이션 방식은 실제 구현 중 조정이 필요하다.

6. 난이도 밸런싱
- danger_level, noise_level, stay_timer, capture_gauge 개념은 확정했다.
- 정확한 증가량, 감소량, 시간 기준은 플레이 테스트로 조정해야 한다.

7. 데이터 세부 조정
- room_objects.lua, rooms.lua, events.lua, monster_nodes.lua의 구조는 설계되었다.
- 실제 구현 시에는 각 방의 최종 연결, hitbox, 더미 리소스 경로를 코드에 맞게 정리해야 한다.

8. Windows exe 패키징
- 최종 산출물 목표는 프로젝트 폴더 + .love 파일 + Windows exe 패키지다.
- exe 패키징은 개발 환경의 LÖVE 버전과 Windows 빌드 방식에 맞춰 진행해야 한다.
