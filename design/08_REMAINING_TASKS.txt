================================================================================
01. 현재 부족한 부분
================================================================================

현재 기획서와 개발 지시서는 스테이지 1을 구현하기 위한 핵심 구조를 대부분 포함하고 있다.
다만 실제 Unity 개발 단계에서 아래 항목은 추가 확정 또는 실제 리소스 작업이 필요하다.

1. 클릭 오브젝트 좌표
- 모든 클릭 오브젝트는 1280 x 720 기준 Rect hit area를 사용하기로 확정했다.
- 하지만 실제 x, y, width, height 값은 최종 방 배경 이미지가 있어야 정확히 정할 수 있다.
- 현재 단계에서는 Unity RectTransform 또는 Serializable Rect 기반 임시 좌표를 사용해야 한다.

2. 실제 방 배경 이미지
- 방 배경 파일명과 아트 방향은 확정했다.
- 실제 이미지 리소스는 아직 없다.
- Codex 구현 시에는 placeholder Sprite로 먼저 실행 가능하게 만들고, 이후 최종 리소스로 교체 가능한 구조가 필요하다.

3. 실제 오브젝트 이미지
- 아이템, 퍼즐 장치, 문, 은신 장소, 몬스터 연출 이미지 파일명은 정했다.
- 실제 이미지 리소스는 아직 없다.
- placeholder Sprite, UI Image, Prefab으로 구현 가능해야 한다.

4. 실제 사운드 파일
- 사운드 파일명과 카테고리는 정했다.
- 실제 ogg 또는 wav 파일은 아직 없다.
- 없을 경우 무음 처리 또는 dummy AudioClip으로 대체해야 한다.

5. 몬스터 등장 연출 세부 프레임
- 몬스터 외형과 등장 방식은 확정했다.
- 각 등장 연출의 Sprite frame 수, 지속 시간, Animator/Timeline/코루틴 방식은 실제 구현 중 조정이 필요하다.

6. 난이도 밸런싱
- danger_level, noise_level, stay_timer, capture_gauge 개념은 확정했다.
- 정확한 증가량, 감소량, 시간 기준은 PlayMode 테스트와 플레이 테스트로 조정해야 한다.

7. 데이터 세부 조정
- RoomDefinition, InteractableDefinition, EventDefinition, MonsterNodeDefinition 구조는 설계되었다.
- 실제 구현 시에는 각 방의 최종 연결, hit area, placeholder resource reference를 Unity serialized asset에 맞게 정리해야 한다.

8. Windows player build
- 최종 산출물 목표는 Unity 프로젝트 폴더 + Windows PC player build다.
- player build는 개발 환경의 Unity 6.3.9f1 설치와 Windows build support 상태에 맞춰 진행해야 한다.

9. Resource-first Harness 설정
- 현재 Unity Assets 안에는 필수 이미지, 오디오, prefab 리소스가 아직 없다.
- 첫 Harness step은 `resource-inventory`여야 한다.
- `resource_manifest.json` 또는 동등한 산출물로 missing / placeholder / final 상태를 먼저 분류해야 한다.
- 이 step이 사용자 승인되기 전에는 C# 시스템, scene, prefab 개발을 시작하지 않는다.

10. 리소스 명칭 불일치 검토
- `03_PUZZLES_ITEMS_EVENTS.txt`의 `fuse_holder`와 `06_RESOURCES_LIST.txt`의 `item_electric_part.png` 명칭이 서로 어긋난다.
- 구현 전 `DesignResourceConsistencyValidation`에서 blocked 처리하고 사용자 결정을 받아야 한다.
