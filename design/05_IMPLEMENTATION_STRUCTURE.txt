02. 데이터 파일 설계 지시
================================================================================

data/items.lua:
- 아이템 10개를 정의한다.
- item id, name, icon, acquired_from, used_in, altar_symbol, flags_on_acquire 포함.
- 현재 실제 아이템 조합식은 비워 둔다.

data/puzzle_inputs.lua:
- 실제 입력 장치를 정의한다.
- study_safe
- laundry_storage_box
- mirror_symbol_panel
- attic_toy_sequence
- master_bedroom_drawer
- basement_altar
- front_door_escape
- 정답값, 성공 이벤트, 실패 이벤트, 보상 아이템, 위험도 증가값 포함.

data/room_objects.lua:
- 각 방의 클릭 오브젝트를 정의한다.
- type 예시:
  - door
  - container
  - item_pickup
  - puzzle_object
  - clue_object
  - hide_spot
  - locked_door
  - monster_spawn_marker
  - escape_door
- 모든 hitbox는 {x, y, w, h} 사각형 구조.
- 최종 배경 리소스가 없으면 임시 좌표를 사용한다.

data/puzzle_chains.lua:
- 목표 체인 9개를 정의한다.
- 각 체인에 rooms, required_items, required_flags, main_steps, completion_condition, next_chains 포함.

data/events.lua:
- 퍼즐 성공/실패, 몬스터 등장, 전기 복구, 지하실 해금, 최종 추격, 클리어, 게임오버 이벤트를 정의한다.

data/rooms.lua:
- 방 ID, 층, 배경, 레이어, 이동 연결, 은신 수, 위험도 보정, 몬스터 진입 여부를 정의한다.

data/monster_nodes.lua:
- 몬스터 이동 노드 그래프를 정의한다.
- 일반 방은 1노드.
- 큰 방/복도/계단/지하실/다락방은 다중 노드.
- 최종 추격 경로를 포함한다.

data/sound_data.lua:
- BGM, ambience, sfx, monster, ui, events 사운드 분류를 정의한다.
- 파일이 없으면 무음 fallback 처리 가능.

data/stage1.lua:
- 스테이지 1 시작 방, 초기 플래그, 클리어 조건, 실패 조건, 진행 초기화 규칙을 정의한다.

================================================================================
03. 핵심 구현 규칙
================================================================================

방 이동:
- 문 오브젝트 클릭 이동
- 화면 가장자리 클릭 이동
- 둘 다 데이터화한다.

클릭 판정:
- 모든 클릭 오브젝트는 사각형 hitbox 사용
- {x, y, w, h}

인벤토리:
- items.lua는 아이템 자체 정의
- room_objects.lua는 어느 방/오브젝트에서 획득하는지 정의

퍼즐:
- puzzle_inputs.lua는 실제 입력 장치
- puzzle_chains.lua는 여러 방을 오가는 목표 체인

몬스터:
- monster_nodes.lua는 방/노드 그래프
- fsm.lua는 상태 전환
- danger.lua는 위험 수치 계산
- monster.lua는 현재 노드, 목표 노드, 추격/수색 처리

저장:
- 진행 저장 없음
- settings.json 저장
- clear_records.json 저장
