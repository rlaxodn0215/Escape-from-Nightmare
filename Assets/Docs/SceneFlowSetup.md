# Scene Flow / Save Load Setup Guide

## Purpose

This setup connects the first playable scene flow:

- TitleScene
- GameScene
- New game
- Continue
- Game over
- Restart from checkpoint
- Ending and return to title

No Scene, Prefab, UI Panel, image, or audio asset is generated by code. Build and UI objects are configured manually in the Unity Editor.

## Build Settings

Add these scenes to Unity Build Settings:

1. `Assets/Scenes/TitleScene.unity`
2. `Assets/Scenes/GameScene.unity`

Scene names must match:

- `TitleScene`
- `GameScene`

If different names are required, update `game_settings.json`:

```json
{
  "settings": {
    "referenceWidth": 1920,
    "referenceHeight": 1080,
    "hideMinSeconds": 5,
    "hideMaxSeconds": 20,
    "titleSceneName": "TitleScene",
    "gameSceneName": "GameScene",
    "ghostDefaultMinArrivalTime": 5,
    "ghostDefaultMaxArrivalTime": 12,
    "ghostDefaultMinLeaveTime": 5,
    "ghostDefaultMaxLeaveTime": 20,
    "ghostDangerThreshold": 1,
    "chaseMoveLimit": 3
  }
}
```

This JSON is an example only. Do not overwrite the real file unless you are intentionally authoring data.

## TitleScene Setup

Recommended hierarchy:

```text
TitleScene
- PersistentManagers
  - GameManager
  - SaveManager
- Canvas
  - TitleMenu
    - NewGameButton
    - ContinueButton
    - DeleteSaveButton
    - QuitButton
    - StatusText
```

Add `TitleMenuUI` to `TitleMenu` and connect:

- `newGameButton`
- `continueButton`
- `deleteSaveButton`
- `quitButton`
- `statusText`

GameManager and SaveManager use DontDestroyOnLoad. If they also exist in GameScene, Singleton will remove duplicates, but the cleaner setup is to keep them in TitleScene only.

## GameScene Setup

Recommended managers:

```text
GameScene
- Managers
  - GameDataManager
  - LocationManager
  - InteractionManager
  - InventoryManager
  - PuzzleManager
  - EndingManager
  - NoiseManager
  - GhostManager
  - HideManager
  - ChaseManager
  - ClueImageManager
- Canvas
  - LocationRoot
  - InventoryBar
  - ClueImagePanel
  - PuzzleUIRoot
  - GameOverPanel
  - EndingPanel
  - HideExitButton
  - GhostStatusPanel
```

GameOverPanel:

- Add `GameOverPanelUI`.
- Connect `rootObject`.
- Connect `messageText`.
- Connect `restartButton`.
- Connect `returnTitleButton`.

EndingPanel:

- Add `EndingPanelUI`.
- Connect `rootObject`.
- Connect `titleText`.
- Connect `messageText`.
- Optionally connect `skipButton`.

EndingManager:

- Connect `endingPanel` to the EndingPanelUI instance.

LocationManager restores `SaveManager.CurrentLocationId` and `SaveManager.CurrentViewId` after GameScene is loaded. If there is no saved position, it falls back to `startingLocationId`, JSON location data, or the first manually registered LocationController.

## Save File Location

SaveManager uses:

```text
Application.persistentDataPath + "/save_data.json"
```

There is one save file only. Multiple save slots are not implemented in this pass.

## New Game Flow

1. Click `NewGameButton`.
2. `TitleMenuUI` calls `GameManager.StartNewGame()`.
3. `SaveManager.ResetDataForNewGame()` creates fresh save data.
4. `SaveManager.SaveGame()` writes the new save file.
5. `GameManager.LoadGameScene(GameStartMode.NewGame)` loads GameScene.
6. `GameManager.FinalizeGameSceneStart()` enters `GameState.Playing`.
7. `LocationManager.ApplySavedPositionOrStartingLocation()` chooses the starting Location/View.

## Continue Flow

1. Click `ContinueButton`.
2. `TitleMenuUI` checks `SaveManager.HasSaveData()`.
3. `GameManager.ContinueGame()` loads GameScene with `GameStartMode.Continue`.
4. `SaveManager.TryLoadGame()` reads `save_data.json`.
5. `LocationManager` restores saved Location/View.
6. Game state becomes `Playing`.

## Game Over Flow

1. `ChaseManager.FailChase()` calls `GameManager.GameOver()`.
2. Game state becomes `GameOver`.
3. `GameOverPanelUI` shows the panel.
4. Restart calls `GameManager.RestartFromCheckpoint()`.
5. Return Title calls `GameManager.ReturnToTitle()`.

## Ending Flow

1. FinalDoor click or puzzle reward calls `GameManager.EnterEnding()`.
2. Game state becomes `Ending`.
3. `EndingManager.PlayEnding()` shows `EndingPanelUI`.
4. After the configured delay, `GameManager.ReturnToTitle()` loads TitleScene.
5. Skip button can call the same return flow early.

## FinalDoor Setup

Direct ending example:

```text
FrontDoor_Button
- Button
- ClickableButton
  - clickableType = FinalDoor
  - requiredItemId = FrontDoorKey
  - linkedPuzzleId = ""
  - linkedDoorId = ""
```

Behavior:

1. Select `FrontDoorKey`.
2. Click the final door button.
3. `InventoryManager.TryUseSelectedItem("FrontDoorKey")` must pass.
4. `GameManager.EnterEnding()` is called.

Final puzzle example:

```text
EntranceDoor_Button
- Button
- ClickableButton
  - clickableType = FinalDoor
  - requiredItemId = FrontDoorKey
  - linkedPuzzleId = Puzzle_EntranceDoor
```

Behavior:

1. Select `FrontDoorKey`.
2. Click the final door button.
3. `Puzzle_EntranceDoor` opens.
4. Puzzle completion uses `rewardType = "Ending"`.
5. `GameManager.EnterEnding()` is called.

## Puzzle Reward Ending

Use this in `puzzles.json` when a puzzle should finish the game:

```json
{
  "puzzleId": "Puzzle_EntranceDoor",
  "locationId": "Entrance",
  "type": "NumberCode",
  "prefabPath": "PuzzleUI/PuzzleUI_EntranceDoor",
  "codeLength": 4,
  "answerVariableName": "EntranceDoorCode",
  "timeLimitSeconds": 0,
  "failCountToNoise": 3,
  "rewardType": "Ending",
  "rewardId": "",
  "requiredItemId": "FrontDoorKey",
  "startsSolved": false
}
```

The example belongs in documentation only. Do not overwrite the real JSON file unless authoring game content.

## Test Scenario

1. Register TitleScene and GameScene in Build Settings.
2. Add GameManager and SaveManager to TitleScene.
3. Connect TitleMenuUI buttons.
4. Press Play from TitleScene.
5. Click New Game and confirm GameScene loads.
6. Confirm LocationManager shows the starting Location/View.
7. Move, pick up an item, or complete a puzzle so SaveManager writes data.
8. Stop Play and start again from TitleScene.
9. Confirm Continue is interactable.
10. Click Continue and verify saved Location/View and state lists are restored.
11. Trigger `ChaseManager.FailChase()` from a test flow or Inspector.
12. Confirm GameOverPanel appears.
13. Click Restart and verify GameScene reloads from save data.
14. Click a FinalDoor or complete a puzzle with `rewardType = "Ending"`.
15. Confirm EndingPanel appears and returns to TitleScene after the delay.

## Notes

- Real scenes are not generated by this implementation.
- UI panels and buttons are not generated by this implementation.
- Existing JSON wrapper files are not overwritten.
- Multiple save slots, cloud save, encryption, final ending cutscenes, and polished game over presentation are still future work.
