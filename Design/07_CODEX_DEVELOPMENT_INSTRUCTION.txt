Escape From Nightmares
Codex Unity 개발 지시서

================================================================================
01. 기본 지시
================================================================================

너는 Unity 6000.3.9f1 기반 Windows PC 게임 개발 AI다.

프로젝트명:
- Escape From Nightmares

목표:
- 첨부된 기획 문서를 기준으로 실행 가능한 Unity 2D 포인트 앤 클릭 호러 게임을 제작한다.

기술 기준:
- Unity 6
- Universal Render Pipeline 2D
- C#
- uGUI
- Unity Input System
- Unity Test Framework

중요:
- 웹 게임이 아니다.
- LÖVE 2D, Lua, Ren'Py, HTML5, JavaScript 기반으로 만들지 않는다.
- Unity 프로젝트 구조와 C# 스크립트 기준으로 구현한다.
- 기획서에 없는 방, 스토리, 캐릭터를 임의로 확장하지 않는다.
- 리소스가 없으면 더미 Sprite/무음 fallback으로 실행 가능하게 만든다.
- 클릭 가능한 오브젝트는 기본 하이라이트하지 않는다.
- 진행 저장은 절대 구현하지 않는다.
- 설정과 클리어 기록만 저장한다.

================================================================================
02. 구현 우선순위
================================================================================

1단계:
- Assets/EscapeFromNightmares 구조 생성
- ScriptableObject 데이터 타입 작성
- GameSession, InventoryService, PuzzleService 작성

2단계:
- GameDirector 작성
- 빈 씬에서도 실행되는 부트스트랩 작성
- 타이틀, 게임 화면, 상태 표시 UI 작성

3단계:
- Stage 1 런타임 데이터 작성
- child_room 시작
- 방 이동 연결
- 아이템 획득

4단계:
- 인벤토리, 지도, 설정 UI 동작
- settings.json 저장
- clear_records.json 저장

5단계:
- 퍼즐 시스템 구현
- 숫자 자물쇠
- 문양 순서
- 색상 순서
- 장난감 순서
- 제단 매칭
- 현관문 탈출

6단계:
- MonsterAIController FSM 구현
- DangerSystem 구현
- HidingSystem 구현
- Game Over 후 child_room 재시작

7단계:
- 실제 Sprite/Audio 리소스 교체 가능한 경로 정리
- RoomController와 Prefab 기반 클릭 영역으로 확장

8단계:
- EditMode 테스트 작성
- PlayMode 테스트 추가
- Windows 빌드 검증

================================================================================
03. 완료 기준
================================================================================

- Start를 누르면 child_room에서 시작한다.
- 인벤토리, 지도, 설정 버튼이 작동한다.
- 방 이동이 가능하다.
- 아이템 획득이 가능하다.
- 주요 퍼즐이 작동한다.
- 몬스터 첫 등장이 발생한다.
- 몬스터 상태가 위험도에 따라 변화한다.
- 은신 시스템이 작동한다.
- capture_gauge가 가득 차면 Game Over 후 처음부터 재시작한다.
- 지하실 제단 퍼즐을 풀면 현관 열쇠를 획득한다.
- 현관 열쇠 획득 후 최종 추격이 시작된다.
- 현관문을 열면 스테이지 클리어된다.
- settings.json과 clear_records.json은 저장 가능하다.
- 진행 저장 파일은 만들지 않는다.
- Windows PC 빌드를 생성할 수 있다.

================================================================================
04. 리소스 관련 지시
================================================================================

이미지:
- 1280x720 기준
- 어두운 손그림풍 2D
- 방 배경, 오브젝트, 전경/후경 레이어 분리 가능
- 파일이 없으면 더미 배경을 사용한다.

사운드:
- AudioClip이 없으면 오류 없이 재생을 생략한다.
- BGM, ambience, SFX, monster, UI 카테고리를 유지한다.

교체 원칙:
- 더미 리소스와 최종 리소스는 같은 경로 규칙을 사용한다.
- 코드가 특정 임시 파일명에 강하게 의존하지 않게 한다.

================================================================================
05. 금지 사항
================================================================================

- LÖVE 2D/Lua 구조로 되돌리지 말 것
- 진행 저장 구현하지 말 것
- 체크포인트 구현하지 말 것
- 클릭 오브젝트 하이라이트를 기본 기능으로 넣지 말 것
- 기획에 없는 방, 스토리, 캐릭터를 추가하지 말 것
- Unity 생성 파일을 수동으로 대량 수정하지 말 것
