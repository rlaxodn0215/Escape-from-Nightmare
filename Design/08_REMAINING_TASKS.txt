================================================================================
01. 현재 부족한 부분
================================================================================

현재 기획과 Unity 개발 지시서는 스테이지 1을 구현하기 위한 핵심 구조를 포함한다.
다만 실제 제작 단계에서 아래 항목은 추가 확정 또는 리소스 작업이 필요하다.

1. 클릭 오브젝트 좌표
- 모든 클릭 오브젝트는 Rect normalizedHitbox를 사용한다.
- 실제 x, y, w, h 좌표는 최종 방 배경 이미지가 있어야 정확히 정할 수 있다.
- 현재 단계에서는 임시 좌표 또는 더미 UI 상호작용 목록을 사용한다.

2. 실제 방 배경 이미지
- 방 배경 파일명과 아트 방향은 확정했다.
- 실제 이미지 리소스는 아직 없다.
- 더미 배경으로 먼저 실행 가능하게 만들고, 이후 Sprite 교체로 반영한다.

3. 실제 오브젝트 이미지
- 아이템, 퍼즐 장치, 문, 은신 장소, 몬스터 연출 이미지 파일명은 정했다.
- 실제 이미지 리소스는 아직 없다.
- Prefab 또는 UI Sprite로 교체 가능한 구조가 필요하다.

4. 실제 사운드 파일
- 사운드 파일명과 카테고리는 정했다.
- 실제 ogg/wav 파일은 아직 없다.
- 없을 경우 무음 처리 또는 재생 생략 fallback을 사용한다.

5. 몬스터 등장 연출 세부 프레임
- 몬스터 외형과 등장 방식은 확정했다.
- 이미지 프레임 수, 지속 시간, 애니메이션 방식은 실제 구현 중 조정한다.

6. 난이도 밸런싱
- danger_level, noise_level, stay_timer, capture_gauge 개념은 확정했다.
- 정확한 증가량, 감소량, 시간 기준은 플레이 테스트로 조정한다.

7. ScriptableObject 데이터 보강
- RuntimeStageFactory의 MVP 데이터를 실제 StageDefinition, RoomDefinition, ItemDefinition, PuzzleDefinition 에셋으로 옮긴다.
- MonsterNodeGraph와 SoundCatalog도 에셋화한다.

8. Windows 빌드
- 최종 산출물은 Unity 프로젝트와 Windows PC 빌드다.
- Unity Build Settings에서 Windows 대상으로 빌드 검증한다.
